# 데이터 베이스
## 정의
 - 체계화하여 통합 관리하는 데이터 저장소
 - 모든 서비스 뒤에 있는 데이터를 저장하고 관리하는 공간
## 데이터 베이스 vs 파일
### 데이터 베이스
 - 조건으로 바로 검색 가능
 - 수천 명이 동시에 이용 가능
 - 권한 및 백업 관리 기능
 - 정해진 구조 / 체계 존재
### 파일
 - 내용을 하나씩 열어봐야 함
 - 동시에 여러 명 사용이 어려움
 - 실수로 삭제할 수 있음
 - 정해진 구조 / 체계가 없음

# SQL
## 정의
 - 구조화된 질의 언어
 - 데이터베이스에게 무엇을 할 지 명령하는 언어
 - 사람과 데이터베이스가 소통하기 위해 필요함
 - 일반적인 영어 키워드를 사용한 직관적 언어

## SQL vs NoSQL
## SQL (Structured Query Language)
 - 정형화 된 데이터를 다루는 데이터베이스에서 사용하는 언어
## NoSQL (non SQL)
 - 정해진 구조 없이 여러 형식을 저장하는 데이터베이스 종류
## SQL DB
 - 엑셀처럼 표 형태의 구조
 - 행과 열이 정해져 있음(2차원)
 - 구조가 엄격함
 - 데이터 규칙이 명확
 - 은행, 학교, 회사 데이터 등에 사용
## NoSQL DB
 - 표 형태가 필요 없음
 - JSON, 문서, 키-값 등 자유로운 구조
 - 구조가 유연함
 - 데이터 형태가 자주 바뀌는 서비스
 - SNS, 게임 등 비정형 데이터 사용처
### Vector DB
 - 데이터를 의미가 있는 숫자 묶음(벡터)으로 저장하는 DB
 - 의미가 비슷한 것들끼리 비슷한 구역에 카테고리 별로 저장
 - AI 검색의 핵심 기술
 - 데이터가 가진 의미의 유사도 기반
 - 데이터간 거리를 통해 데이터 탐색
 - 검색 시스템 및 챗봇에 주로 사용
 - 빠른 검색에 유용
 - 임베딩 : 데이터를 컴퓨터가 처리하기 좋은 숫자 벡터 형태로 변환
### Graph DB
 - 데이터를 표가 아닌 점과 선으로 저장하는 데이터베이스
 - 기존 SQL DB는 정리와 기록에 특화 => 2개 이상의 데이터 사이의 관계에 강한 DB가 필요해짐
 - 데이터가 가진 관계 기반
 - 데이터간 관계를 통해 데이터 탐색
 - 네트워크 기반 데이터에 주로 사용
 - 다중 연결 정보 저장에 유용
 - ex) SNS 친구추천, 노선도로 최단 경로 찾기, 어느 분야의 전문 지식 등
